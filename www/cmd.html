Getidmis, the INN IDMIS Downloader

Command line installation and usage

If you are happy using the command line, then you can simply download the getidmis program and run it.

 Windows   Linux   Mac 

Note, depending on your browser, you may obtain an alert saying something like 'Insecure download blocked'. Click the keep option.

With Chrome or Edge browsers, the download may get 'stuck' as a Unconfirmed xxxxx.crdownload file. If this is the case, click the download progress button and click the Keep button. Alternatively, go to the folder where you have downloaded it and rename the file.

Note: The dire warnings about 'insecure downloads' really can be safely ignored! They are not suggesting any security issues with the file itself, they are simply saying that it is downloaded over HTTP rather than HTTPS (which encrypts the file as it is being downloaded).

As usual, you can put the executable program somewhere in your path, or run it in the current directory.

Usage

getidmis [-c cert.p12] [-f passwordfile] [-p password] [-x curlPath] reqnum [reqnum ...]
         -c Specify the certificate file
         -f Specify a file containing the password
         -p Specify the password
         -x Specify the full path to curl

To use getidmis, you need to provide the certificate file and the certificate password. You have five options of how to do this:

  1. Specify the certificate file using the -c flag and the password using the -p flag.
  2. Create a file containing the password, then specify the certificate file using the -c and the file containing the password using the -f flag.
  3. Ensure the certificate file is called cert.p12) and create a file containing the password called certpw.txt). If these are in your current directory, you do not need to specify them on the command line.
  4. Set the environment variable IDMISCERTFILE to specify the path to your certificate file. Create a file containing your certificate password and set the environment variable IDMISPWDFILE to specify the path to your password file. Obviously this should be done in your startup file (.profile, .bashrc, .tcshrc, etc.)
  5. Set the environment variable IDMISCERTFILE to specify the path to your certificate file and set the environment variable IDMISPASSWD to specify your password. Again, this should be done in your startup file.

If both the password file and the actual password are specified, the actual password takes precedence. i.e.:
a provided password takes precedence over the password file
--and--
values on the command line take precedence over environment variables, which take precedence over defaults.
